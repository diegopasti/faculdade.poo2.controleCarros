Controle de Carros


Descrição

O projeto Controle de Carros foi desenvolvido com o intuito de implementar um exemplo em Python onde utilizássemos os padrões comportamentais Mediador, Memento e Visitador. Contudo, devido dificuldades no entendimento do padrão visitador somente os dois primeiros foram implementados.

Padrões de Projeto

Mediador: Esse padrão foi implementado para simular um controle de componentes do carro, sincronizando o funcionando dos mesmos evitando que o funcionamento de um componente pudesse interferir em outro componente. Isso é feito bloqueando o funcionando de um componente enquanto outro esta sendo usado evitando algum tipo de problema.

Memento: Esse padrão foi implementado para controlar o estado do velocímetro do carro sem interferir no encapsulamento das classes responsáveis pelo velocímetro. Como ele é possivel registrarmos um determinado estado do velocimetro e restaura-lo posteriormente.

Visitador: Não foi implementado visto as dificuldades para entende-lo.

Diferenças de Implementação

As diferenças de implementação em Python e em Java da-se puramente por questões de sintaxe, não há dificuldade de implementar em python os padrões acima citado. Nesse exemplo ainda procurei manter as classes um pouco mais separadas em arquivos diferentes para alegria dos programadores java, mas elas poderiam estar tudo em apenas um arquivo que é um recurso da linguagem pyhton. Ainda utilizei em partes o padrão de nomenclatura java para ajudar. Aos programadores Java, só faltou publics, privates.. e declarações de atributos.. mas como já disse são os poucos aspectos que diferenciam essa implementação.
